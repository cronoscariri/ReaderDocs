Introduction Last updated: 2022-02-26

This is just a Web Socket interface for RFID reader built in Java from the Octane SDK.

Reader RFID

Just tested with the Impinj Speedway Revolution reader. These readers are known to work well with sllurp, but it should be adaptable with not much effort to other LLRP-compatible readers:

  • Impinj Speedway (R1000)
  • Impinj Speedway Revolution (R220, R420)
  • Impinj Speedway xPorta
  • Motorola MC9190-Z (handheld)

Features

See some features that this interface supports. Other functionality can be added later.

Connection Connect and disconnect to an RFID reader.
Read Start, stop, resume reading tags.
Search Mode Single Target.
Antenna status Antenna activation and deactivation.
Antenna power Antenna power regulation.
Read time Takes time directly from the reader..

Architecture

The architecture used in this approach is a micro service, where you has a service, available to be consumed by an application built in any programming language.

Config Reader

To get started you must have a wired network configured with static IP.

Note

The configuration shown was performed on Linux Ubuntu.

Config Network

Settings >> Network New Wired Network

  • IP Address: 192.168.0.98
  • Network Mask: 255.255.255.0
  • Default Route: 192.168.0.1
  • Broadcast Address: 192.168.0.255

Access CGI Impinj

Open the browser and access the IP you set for the Impinj reader.
Enter your login and password.

By default:

  • URL: your IP reader
  • User: root
  • Password: impinj

Source code

If you are a developer and would like to edit the source code for your specific needs, this is the place.

Requirement

What you need before you start.

  • Java JRE and SDK => 7
  • IDE Eclipse or any other of your choice.
Clone project

Clone the project to your local development environment and make any changes you want.

git clone https://github.com/cronoscariri/ReaderWebSocket.git

Se você não sabe como começar, de uma olhada na documentação java doc do Octane SDK do Impinj Reader.

Note

After making all the changes you want, generate a binary of your WebSocket microservice.
Eclipse >> Project >> Export >> Java >> Runnable JAR file.

Run bin

To run your binary file, run the command below:

java -jar filename.jar 6868

Note

You can set any value for the port parameter.

Client CLI

Command Line Interface is useful for testing applications quickly, it was with this in mind that we have this client where you can connect to Reader by just sending text messages on the command line.

User guide

Download

Baixe o binário cli_client.bin executável.



Run application client CLI
  • Hostname localhost or address IP computer.
  • Port: Port open computer.

Para executar o binário, execute o comando abaixo:

java -jar cli_client.jar localhost 8686

Send messages
Connect reader { "command": "connect_reader" }
Connect reader custom {"command": "connect_reader", "hostname": "192.168.0.99"}
Start reader {"command": "start_reader"}
Stop reader {"command": "stop_reader"}
Disconnect reader {"command": "disconnect_reader"}
Disconnect client {"command": "disconnect_client"}

Integrations

If you want to know how to integrate your application with the WebSocket microservice this is the place.

Application client

For you to integrate your application with microservice, you need to create a connection to the WebSocket in your client application. Okay, done that, everything is ok, now it's spo to send and receive messages in JSON format. In case you didn't understand how to do this, see the example of Desktop, Web and Mobile client applications.

Message Json

Write your message in json, see some examples of parameters available to configure.

Note

When any parameter is omitted, the reader will use the parameter from the manufacturer's default configuration.

This only applies to: {"command": "connect_reader"}
